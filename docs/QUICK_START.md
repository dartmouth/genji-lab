## üéì Student Quick Start

### Goal: Create your first annotation in 5 minutes

#### Step 1: Log In (1 minute)

**Option A: Institutional Login (CAS)**
1. Go to your Genji URL
2. Click "Login with CAS"
3. Enter your school credentials
4. ‚úÖ You're in!

**Option B: Local Account**
1. Go to your Genji URL
2. Click "Register"
3. Fill in: username, email, password, name
4. Click "Create Account"
5. ‚úÖ Account created!

#### Step 2: Join Your Classroom (1 minute)

1. Get your classroom code from your instructor (looks like: `ENG101-FALL2025-ABC123`)
2. Click **"Join Classroom"** or the **+** button
3. Enter the code
4. Click **"Join"**
5. ‚úÖ You're in the classroom!

**Can't find the button?** Look for "My Classrooms" in the navigation, then "Join New Classroom"

#### Step 3: Open Your First Document (1 minute)

1. Click on your classroom name
2. You'll see document collections (e.g., "Week 1 Readings")
3. Click on a collection
4. Click on any document title
5. ‚úÖ Document opens!

#### Step 4: Create Your First Annotation (2 minutes)

1. **Select text** you want to comment on:
   - Click and drag your mouse over the text
   - The text will highlight
   
2. **Choose annotation type**:
   - Right-click or click the annotation button that appears
   - Select **"Comment"** (for your first one)
   
3. **Write your annotation**:
   - Type your observation, question, or analysis
   - Keep it focused and specific
   
4. **Submit**:
   - Click **"Save"** or **"Submit"**
   - ‚úÖ Your first annotation is live!

**See your annotation:** It should appear in the sidebar or next to the text you selected.

## üë®‚Äçüè´ Instructor Quick Start

### Goal: Set up a classroom and upload your first document in 10 minutes

#### Step 1: Get Instructor Access (varies)

If you don't already have instructor permissions:
1. Contact your system administrator
2. Request **"Instructor"** role
3. Wait for confirmation (may be immediate or take a day)
4. ‚úÖ Once granted, you'll see "Admin" or "Instructor Panel" in your navigation

**Already have access?** Skip to Step 2!

#### Step 2: Create Your Classroom (2 minutes)

1. Click **"Admin"** or **"Instructor Panel"** in the navigation
2. Click **"Manage Classrooms"**
3. Click **"Create New Classroom"** or the **+** button
4. Fill in:
   - **Name**: e.g., "English 101 - Fall 2025"
   - **Description**: Brief course description (optional)
5. Click **"Create Classroom"**
6. ‚úÖ Classroom created!

**Important:** Copy your **classroom code** - you'll share this with students!

#### Step 3: Upload Your First Document (3 minutes)

**Before you start:** Have a .docx (Word document) file ready. Only .docx format is supported.

1. Go to **Admin Panel** ‚Üí **"Manage Documents"** or **"Upload Document"**
2. Click **"Upload Document"** or the **+** button
3. Fill in:
   - **Title**: Document name (e.g., "Week 1 - Chapter 1")
   - **Collection**: Create new or select existing
   - **Classroom**: Select your classroom
4. Click **"Choose File"** and select your .docx file
5. Click **"Upload"** or **"Import"**
6. Wait for processing (10-30 seconds for most documents)
7. ‚úÖ Document uploaded and available!

**Verify it worked:**
- Go to your classroom view
- Check the collection - your document should appear

#### Step 4: Add Students (2 minutes)

**Easy method - Share classroom code:**
1. Find your classroom code (in classroom settings)
2. Share it with students via:
   - Course LMS (Canvas, Blackboard, etc.)
   - Email
   - Course syllabus
3. Students use the code to join automatically
4. ‚úÖ Students join on their own!

**Manual method - Add students yourself:**
1. Go to your classroom
2. Click **"Manage Members"**
3. Search for students by name/email
4. Select and click **"Add"**
5. ‚úÖ Students added!

#### Step 5: Share with Students (1 minute)

Send your students:
1. **Genji URL**: [Your institution's Genji link]
2. **Classroom code**: Your unique code
3. **Instructions**: "Join the classroom and read Week 1 documents"
4. **Student Guide**: Share link to [Student Guide](guides/STUDENT_GUIDE.md)

‚úÖ **Your classroom is ready!**

## üîß Administrator Quick Start

### Goal: Create your first user and assign roles in 5 minutes

#### Step 1: Access Admin Panel (1 minute)

1. Log in to Genji with your admin account
2. Click **"Admin"** or **"Admin Panel"** in navigation
3. You should see:
   - User Management
   - Classroom Management
   - System Settings
   - Content Moderation
4. ‚úÖ Admin panel open!

**Don't see Admin Panel?** Contact the system owner to grant you administrator role.

#### Step 2: Create a User (2 minutes)

1. Click **"User Management"** or **"Users"**
2. Click **"Create New User"** or the **+** button
3. Fill in:
   - **Username**: e.g., "jsmith"
   - **Email**: User's email address
   - **First Name**: e.g., "Jane"
   - **Last Name**: e.g., "Smith"
   - **Password**: Temporary password (or they use CAS)
   - **Role**: Select "student" for now
4. Click **"Create User"**
5. ‚úÖ User created!

**Best practice:** Use institutional email addresses (e.g., `@university.edu`)

#### Step 3: Assign Instructor Role (1 minute)

**To make someone an instructor:**

1. In **User Management**, find the user
2. Click on their name to open profile
3. Find the **"Roles"** section
4. Check **"instructor"** (keep "student" checked too)
5. Click **"Save Changes"**
6. ‚úÖ User is now an instructor!

**What changes:**
- User can now create classrooms
- User can upload documents
- User sees "Admin Panel" or "Instructor Panel"

#### Step 4: Verify Everything Works (1 minute)

1. Have the new user log in
2. They should see their new permissions
3. Instructor should see "Admin Panel"
4. ‚úÖ Success!

## üíª Developer Quick Start

### Goal: Run Genji locally in 10 minutes

#### Prerequisites

Install these if you don't have them:
- **Docker** and **Docker Compose** ([Download](https://www.docker.com/products/docker-desktop))
- **Git** ([Download](https://git-scm.com/))

#### Step 1: Clone the Repository (1 minute)

```bash
# Clone the repo
git clone https://github.com/dartmouth-itc/rc-genji.git
cd rc-genji
```

‚úÖ **Repository cloned!**

#### Step 2: Start with Docker Compose (2 minutes)

```bash
# Start all services (database, backend, frontend)
docker compose up
```

**What's happening:**
- PostgreSQL database starting on port 5432
- FastAPI backend starting on port 8000
- React frontend starting on port 5173
- First run takes 2-5 minutes (downloading images, building)

**Watch for:**
```
‚úì Container rc-genji-db-1       Started
‚úì Container rc-genji-api-1      Started
‚úì Container rc-genji-ui-1       Started
```

‚úÖ **All services running!**

#### Step 3: Access Genji (30 seconds)

Open your browser:

- **Frontend**: http://localhost:5173
- **API Docs**: http://localhost:8000/docs
- **API Health**: http://localhost:8000/health

‚úÖ **Genji is running!**

#### Step 4: Create Your First Admin User (2 minutes)

**Option A: Using the API directly**

1. Go to http://localhost:8000/docs
2. Find **POST /users/register**
3. Click **"Try it out"**
4. Fill in the JSON:
```json
{
  "username": "admin",
  "email": "admin@example.com",
  "password": "admin123",
  "first_name": "Admin",
  "last_name": "User"
}
```
5. Click **"Execute"**
6. ‚úÖ User created!

**Option B: Using the UI**

1. Go to http://localhost:5173
2. Click **"Register"**
3. Fill in the form
4. Click **"Create Account"**
5. ‚úÖ User created!

**Make yourself an admin:**
```bash
# Connect to the database
docker compose exec db psql -U postgres -d genji

# Grant admin role (replace 'admin' with your username)
UPDATE users SET is_admin = true WHERE username = 'admin';

# Exit
\q
```

‚úÖ **You're now an admin!**

#### Step 5: Verify Everything Works (2 minutes)

**Test checklist:**

1. ‚úÖ Can you log in? (http://localhost:5173)
2. ‚úÖ Can you access admin panel?
3. ‚úÖ Can you create a classroom?
4. ‚úÖ Can you see API docs? (http://localhost:8000/docs)

**All working?** üéâ **Success!**

## üÜò Troubleshooting

### Common Issues - All Users

**Problem: Can't log in**
- ‚úì Check username/password are correct
- ‚úì Try "Forgot Password" if using local auth
- ‚úì Verify account exists and is active
- ‚úì Try different browser or incognito mode

**Problem: Classroom code doesn't work**
- ‚úì Double-check the code (copy-paste to avoid typos)
- ‚úì Verify classroom is active
- ‚úì Make sure you're logged in
- ‚úì Contact instructor for new code

**Problem: Document won't load**
- ‚úì Refresh the page
- ‚úì Check internet connection
- ‚úì Try different browser
- ‚úì Contact instructor or admin

**Problem: Annotation not saving**
- ‚úì Check internet connection
- ‚úì Make sure you clicked "Save" or "Submit"
- ‚úì Try logging out and back in
- ‚úì Try different browser

### Common Issues - Instructors

**Problem: Can't upload document**
- ‚úì Verify file is .docx format (not .doc or .pdf)
- ‚úì Check file isn't corrupted (open in Word first)
- ‚úì Try a smaller file first
- ‚úì Contact administrator if persists

**Problem: Students can't see my classroom**
- ‚úì Verify you shared the correct classroom code
- ‚úì Check classroom is set to "active"
- ‚úì Make sure students are logged in
- ‚úì Manually add students if needed

### Common Issues - Developers

**Problem: Docker won't start**
- ‚úì Check Docker Desktop is running
- ‚úì Check port 5432, 8000, 5173 aren't in use
- ‚úì Try `docker compose down` then `up` again
- ‚úì Check Docker has enough resources allocated

**Problem: Database connection error**
- ‚úì Wait 10-15 seconds after starting (DB needs time)
- ‚úì Check `docker compose logs db` for errors
- ‚úì Verify database container is running: `docker compose ps`

**Problem: Frontend can't reach API**
- ‚úì Check API is running: http://localhost:8000/health
- ‚úì Check browser console for CORS errors
- ‚úì Verify API_URL in frontend config

**Problem: "Module not found" errors**
- ‚úì Rebuild containers: `docker compose build`
- ‚úì Install dependencies if running locally
- ‚úì Check you're in the right directory
