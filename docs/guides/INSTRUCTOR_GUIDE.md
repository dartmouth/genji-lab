### Adding Students to a Classroom

[Multiple methods for adding students]

#### Method 1: Classroom Code/Join Link

[Instructions for sharing invite code]

1. Open your classroom
2. Find the "Classroom Code" or "Invitation Link"
3. Share this code with students via:
   - Course LMS
   - Email
   - Course website
4. Students use the code to join automatically

#### Method 2: Manual Addition

[Instructions for manually adding students]

1. Navigate to your classroom
2. Click "Manage Members" or "Add Students"
3. Search for students by:
   - Username
   - Email
   - Name
4. Select students to add
5. Click "Add Selected Students"

#### Method 3: Bulk Import (if available)

[Instructions for importing student lists]

1. Prepare a CSV file with student information
2. Navigate to classroom settings
3. Click "Import Students" or "Bulk Add"
4. Upload your CSV file
5. Review and confirm additions

### Uploading Documents

[Step-by-step instructions for uploading]

**Supported Format:** Word Documents (.docx)

**Upload Process:**

1. Navigate to Admin Panel > Manage Documents
2. Click "Upload Document" or "+" button
3. Fill in document information:
   - Title
   - Description (optional)
   - Select collection to add to
4. Click "Choose File" and select your .docx file
5. Click "Upload" or "Import"
6. Wait for processing (may take a moment for large documents)
7. Confirm successful upload

**What Happens During Upload:**
- Document is parsed into paragraphs/elements
- Text formatting is preserved
- Document becomes available for annotation
- Students in associated classroom can access it

**Document Upload Tips:**
- Use well-formatted Word documents
- Check formatting after upload
- Test with a single document first
- [Other tips based on your experience]

## Common Workflows

### Workflow 1: Creating a Weekly Reading Assignment

[Step-by-step workflow]

1. Create a new collection: "Week [X] Readings"
2. Upload documents for the week
3. Assign collection to your classroom
4. Create an announcement or assignment in your LMS
5. Monitor student annotations throughout the week
6. Provide feedback and facilitate discussion
7. Review participation before next week

### Workflow 2: Facilitating a Class Discussion

[Using annotations for discussion]

1. Assign a reading with specific annotation prompts
2. Students annotate individually (due date 1)
3. Students respond to classmates (due date 2)
4. Review and highlight interesting threads
5. Use annotations to guide in-class discussion
6. Follow up with summary or reflection

### Workflow 3: Document Comparison Activity

[If supported]

1. Upload multiple versions or related texts
2. Set up comparison view
3. Prompt students to identify similarities/differences
4. Students annotate points of comparison
5. Review comparative analysis annotations
6. Discuss findings as a class

### Workflow 4: Peer Review Process

[Using Genji for peer feedback]

1. Students annotate their assigned texts
2. Instructor creates peer review groups
3. Students read and respond to peer annotations
4. Instructor reviews both original and peer feedback
5. Provide meta-feedback on the review process
